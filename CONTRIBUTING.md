# Contributing to MapMyCode

## Development flow

1. Fork the repository.
2. Create a branch for your change.
3. Run `npm install`.
4. Run `npm run build` before opening a pull request.
5. Include screenshots or a short demo when changing the UI.

## Pull request expectations

- Keep changes focused.
- Explain the problem and the approach.
- Note tradeoffs or known limitations.
- Add or update documentation when behavior changes.

## Issue quality

Good issues include a minimal reproduction, expected behavior, actual behavior, and environment details.
